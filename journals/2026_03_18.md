	- ## **Agenda**

	- ---
	- ## **Discussed**
		- ### **1. Background: AWS Now Bills Ingram From Two Entities Per Country**
			- Starting early **2026**, AWS has changed its billing structure:
				- **AWS Inc. (Global Entity)**
				  → Always bills **in USD**
				- **AWS Local Entity** (e.g., AWS Canada, AWS Mexico, AWS Australia)
				  → Bills in **local currency**
				- These represent **two different billing streams**:
					- **Infrastructure‑as‑a‑Service (IaaS)** base consumption
					- **AWS Marketplace** transactions (third‑party vendors)
			- Tim clarified:
				- AWS Marketplace 3rd party vendors bill AWS in *their own currency*, and AWS passes that currency directly to Ingram **without FX conversion**.
				- Ingram must therefore *also* pass through the same currency to the reseller to avoid FX risk.
				- Marketplace charges may arrive in: USD, EUR, GBP, CAD, etc.
				- This creates **multi‑currency usage lines** for the same customer & same subscription.
	- ---
		- ### **2. Connect: Capability to Handle Multi‑Currency Usage Files**
			- Mike & Diego confirmed:
				- The AWS usage files use the **TR schema**.
				- Connect *supports custom vendor fields in TR schema.
				- Connect **can ingest multiple files per marketplace**, each with a different currency.
				- Connect **can assign different currencies** per usage file
			- Therefore:
				- **No new Connect development is required.**
				- Connect can:
					- Accept *two usage files* (USD + local currency)
					- Pass custom vendor attributes downstream
					- Split files if needed
			- However, one blocker exists:
				- ### ❗ CloudCheckr only delivers usage data in USD
				- Even when AWS provides multiple currencies
				- Meaning the upstream CloudCheckr integration is broken
				- This prevents Connect from receiving correct currency lines
				- This is a **major dependency** → to be discussed with CloudCheckr owner.
		- ---
		- ### **3. XVS Usage Collector & Rating Engine Behavior**
		- Felipe performed a live test:
			- Replayed the same AWS usage file twice:
				- Once in **USD**
				- Once in **CAD**
			- Usage Collector accepted both files
			- Rating Engine created correct usage charges
			- Charges were stored in BSS with correct currencies
			  XVS **can ingest & process multi‑currency charges** for the same subscription.
			- But…
				- ### ❗ Billing Orders Fail
				- XVS tries to create Billing Orders in SAP
				- It **attempts FX conversion** when account currency ≠ usage line currency
				- FX rates missing → BO creation fails
				- FX conversion itself is **NOT acceptable** (would introduce risk)
		- **Tim emphasized:**
		  *“We must NEVER convert FX for marketplace pass‑through charges.”*
		- ---
		- ### **4. The Core Blocker: Reseller Accounts Per Currency**
			- Resellers may only have **one account** (e.g., CAD).
				- AWS Marketplace lines may require **USD**, EUR, GBP…
			- Therefore:
				- XVS must determine **the correct reseller account per currency**.
				- If reseller lacks the required currency account → error or fallback scenario needed.
				- SAP must receive charges under **the correct reseller PCN per currency**.
				- Export Engine + Tipco must route charges by currency without conversion.
				- ### This requires  **architecture-level enhancements** :
					- Mapping rules: reseller account → allowed currencies
					- Account discovery logic
					- Error handling for unsupported currency
					- Modified invoice pipeline to avoid FX attempts
					- Possibly new middleware logic between XVS → SAP
		- ---
		- ### **5. System Flow Gaps Identified**
		  
		  | Layer | Status | Required Changes |
		  | **CloudCheckr** | ❌ Always sends USD | Must support original AWS currency |
		  | **Connect** | ✔ Multi-currency OK | None |
		  | **Usage Collector** | ✔ Accepts data | None |
		  | **Rating Engine** | ✔ Creates correct charges | None |
		  | **XVS Billing Order Creation** | ❌ Attempts FX conversion → fails | Must support multi‑currency billing accounts |
		  | **SAP Billing** | ⚠ Needs correct account mapping | Must receive correct document currency |
