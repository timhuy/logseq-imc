	- ## Discussed
		- ### Project Context
			- **Goal**: Migrate existing Autodesk subscriptions into XBS 3.0 BEFORE renewal orders arrive  
			  **Scale**: ~10,000 subscriptions (10K for the 9 resellers)  
			  **Business Model**: No Ingram billing/invoicing (MSRP=Price, margin=0)  
			  **Orders**: Never placed through Ingram - pure subscription data migration  
			  **Automation**: Manual initial load → daily automation post go-live  
			  **First Time**: Autodesk team never done this migration exercise before
		- ### End-to-End Migration Flow
			- ### CDR File Structure (Excel in SharePoint)
				- **6 Sheets Total** (focus on 2 for data entry):  
				  **LVL1 Subscription Header** ← Your main data entry sheet  
				  **LVL2 Subscription Items** ← Line items (multiple rows per sub)  
				  Credit LBL, Product PM1, Mock sub (XPS R&D managed)
				- ### Level 1: Subscription Header (1 row per subscription)
				  **MANDATORY FIELDS** (from 9 reseller exports):  
				  ├── Vendor Subscription ID (unique reconciliation key)  
				  ├── Subscription Offer Name (XPS service plan)  
				  ├── Start Date & Expiration Date  
				  ├── Billing Model: [Prepaid Term | Annual | Monthly | Quarterly]  
				  ├── Billing Period Length (e.g., 3.0 for quarterly)  
				  ├── Billing Period Unit: [Month | Year]  
				  ├── Billing Schedule: [Advance | Arrears]  
				  
				  **INGRAM MICRO FIELDS** (static/batch copy):  
				  ├── Contact: name/email/phone  
				  ├── Reseller: A2, BCN numbers (internal use only)  
				  ├── Bill-to: 3-digit code (NET30/45/60/75)  
				  ├── Payment: 3-digit code or 998 (credit card)  
				  
				  **AUTODESK SPECIFICS**:  
				  ├── Cancelable: FALSE (all subscriptions)  
				  ├── No order/PO data needed  
				  ├── No billing period start date (not used)  
				  ├── Grace period: None from vendor
				- ### Level 2: Subscription Items (multiple rows per subscription)
				  **STRUCTURE**: Repeat Vendor Sub ID for each MPN/item  
				  **REQUIRED FIELDS**:  
				  ├── Vendor Subscription ID (links to LVL1 - repeat X times)  
				  ├── Product MPN ← Product Catalog lookup (OperatorID mapping)  
				  ├── Product Name ← Product Catalog  
				  ├── Quantity ← Reseller export  
				  ├── MSRP ← Catalog (same as Price for Autodesk)  
				  ├── Price ← Same as MSRP (margin=0)  
				  ├── Currency ← USD (standard)  
				  ├── Margin ← 0 (no Ingram margin)  
				  ├── Pricing Term Length ← 12 months (from subscription data)  
				  
				  **ITEM TYPES** (if known): BPU = Usage-based | RES = Reservation-based
				- ### Data Sources & Mapping Process
				  **Primary Data**: 9 reseller Excel exports (~10K subs each)  
				  **Lookup Table**: Product catalog (XPO→Connect auto-sync)  
				  Maps Autodesk OperatorID → Ingram MPN/Product Name  
				  **Static Fields**: Batch copy across all rows (billing model, cancelable=FALSE)
				  
				  **Process**:  
				  1. LVL1: Copy subscription headers from reseller files  
				  2. LVL2: For each sub → lookup OperatorID in catalog → get MPN → add row  
				  3. Repeat Vendor Sub ID + populate quantity/pricing per line item
				- ### File Strategy Decision
				  **FINAL**: 9 separate CDR files (1 per reseller) ✅  
				  ├── Enables parallel team work (1 person = 1 reseller)  
				  ├── Better volume management (~10K subs/file)  
				  ├── Tracks reseller-specific vendor updates  
				  ├── Mock→Production batch processing (reseller by reseller)
